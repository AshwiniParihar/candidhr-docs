## Opening the Editor

From the Workflows list, click a workflow's **name** to open it in the editor. The editor tab is selected by default.

<Frame>
  <img src="/images/workflows/image71.png" alt="Workflow editor canvas" />
</Frame>

The canvas opens with an **Initial Stage** node that represents the entry point — all candidates entering via the job application form or email invite start here.

---

## Editor Navigation

The workflow header has three tabs:

| Tab | Description |
|---|---|
| **Editor** | Visual canvas where you build and configure the workflow |
| **Logs** | Execution history — see what happened and when |
| **Versions** | Manage and activate workflow versions |

You can also switch between workflow versions using the **version selector** dropdown in the header.

---

## Adding Stages

Click the **+ (Add Block)** button below any node to open the **Module Modal**. The modal has three tabs:

### Modules

Pipeline stages that process candidates:

| Module | Description |
|---|---|
| **Resume Screening** | AI-scores resumes against the job's criteria. Can only be added once. |
| **Assessment** | Sends a test to the candidate and tracks results |
| **Automated Video Interview** | Sends a video question set; candidate records responses |

### Conditions

Conditional branching based on stage results:

| Condition | Available Criteria |
|---|---|
| **Resume Screening** | Overall score, experience, current CTC, notice period, location |
| **Assessment** | Status (Pass / Fail / Completed) and percentage score |
| **Automated Video Interview** | Link Opened, Started, Completed, Disqualified |

Conditions split the workflow into **True** and **False** branches. Add stages under each branch to handle both outcomes.

<Note>
  Conditions cannot be added directly below the Initial Stage. At least one module must exist before adding a condition.
</Note>

### Actions

Automation steps that run at a stage:

| Action | Description |
|---|---|
| **Notify Recruiter** | Sends an in-app or email notification to the recruiter |
| **Update Status** | Automatically changes the candidate's application status |
| **Add a Note** | Attaches a note to the candidate's profile |
| **Notify Applicant** | Sends a message to the candidate |
| **Send SMS** | Sends an SMS to the candidate |

## Configuring Stages

Click any node on the canvas to open the **Right Panel** and configure that stage.

### Assessment Configuration

<Frame>
  <img src="/images/workflows/image78.png" alt="Assessment stage configuration" />
</Frame>

1. Select a pre-created assessment from the dropdown
2. Choose which questions to include
3. Set the **Valid Till** period — how many days the candidate has to complete it

### Automated Video Interview Configuration

<Frame>
  <img src="/images/workflows/image80.png" alt="Video interview stage configuration" />
</Frame>

1. Select a pre-created video interview question set
2. Add or remove individual questions
3. Set the validity period
4. Save the configuration

## Configuring Conditions

<Frame>
  <img src="/images/workflows/image73.png" alt="Condition configuration panel" />
</Frame>

In the Right Panel for a condition node:

1. Choose **AND** (all rules must match) or **OR** (any rule must match)
2. Add one or more rules — each rule selects a field, an operator, and a value
3. Add stages under the **True** branch (candidate meets the condition) and the **False** branch (candidate does not)

**Example — Resume Screening condition using AND logic:**

- Overall Score **≥ 6**
- Notice Period **≤ 15 Days**
- Expected CTC **≤** budgeted salary

---

## Deleting Stages

| Deletion Type | Behaviour |
|---|---|
| **Delete** (regular stage) | Removes the stage directly |
| **Delete All** | Removes the condition node and all stages in both branches |
| **Keep True Branch** | Removes the condition and false branch; true branch stages remain |

<Warning>
  Stages cannot be deleted if there are active candidates currently at that stage.
</Warning>
